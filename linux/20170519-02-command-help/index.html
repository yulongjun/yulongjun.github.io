02-查看命令的帮助

发表于 2017-05-19 | 分类于 linux | 阅读次数

whatis

模糊操作命令的帮助，并列出在哪个手册，比如下面搜索date，会列出条目和简介里带date字眼的所有命令字段。前面是命令，后面是简介。命令后面括号里的数字指的就是man手册的序号。

执行的whatis 命令是查询数据库提供的，并不一定是最新的，可手动更新数据库，第一次运行可能会有点慢：

  • CentOS 6 上：makewhatis
  • CentOS 7 上：mandb

help & --help

大部分命令都有--help选项，也是一个简要的帮助文档。

man

man后的格式和内容

详细的帮助文档：man COMMAND

  • NAME 功能能行说明
  • SYNOPSIS 语法概要
    • []表示可选选项
    • <>表示必需提供的
    • |多选一
    • {}分组的
    • ...前面的内容可以出现多次
  • DESCRIPTION 描述
  • OPTIONS 选项
  • EXAMPLES 使用示例
  • AUTHOR 作者
  • REPORTING BUGS 报告bug方式
  • COPYRIGHT 版权
  • SEE ALSO 参考其他

man的序号 1-9

  1. 用户命令
  2. 系统调用
  3. 库调用
  4. 设备文件
  5. 文件格式
  6. 游戏
  7. 杂项
  8. 管理命令

whatis COMMAND：可以查看命令有哪几个manual

默认查的是序号最小的那个man手册，如要要查其它的，可以用man 序号 COMMAND来查看命令,例如man 5 tar

man手册操作方法

翻屏操作：

很多特别像vim的命令

键盘按键 效果
空格键 向下滚动一屏
b 向上滚动一屏
j 向上移动一行
k或回车 向下移动一行
gg 跳转至第一行
5g 跳转到第5行
G 跳转到最后一行
q 退出

文本搜索

键盘按键 效果
/keyword 向下查找，不区分大小写
?keyword 向上查找，不区分大小写
n 查找下一个，与查找方向相同
N 查找上一个，与查找方向相反

man选项：
man -M /path/to/somedir： 某些程序man目录不在标准的/usr/share/man到指定目录下查找命令手册并打开它。

程序自带帮助文档

很多会放在/usr/share/doc/APP-VERSION下

  • README：程序的相关信息
  • INSTALL：安装帮助
  • CHANGES：版本改动信息

http://www.redhat.com/doc
