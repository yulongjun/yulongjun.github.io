更改提交的操作

1. git reset（回溯历史版本）

看上面最后一个git log --graph的图标，我们要把先回溯到feature-A分支创建前，然后创建一个fix-B分支。

  • 回退到“Add index”时候的状态，用git reset --hard 哈希值
  • 创建fix-B分支
    创建特性分支fix-B。
  • 修改READ.md文件，添加一行- fix-B。
  • 提交READ.md文件
  • 推进至feature-A分支合并后的状态

git log 只能查看当前状态为终点的历史日志。要查看整个仓库的所有操作日志，要用git reflog，找到回溯历史之前的哈希值，通过git reset --hard来恢复到回溯之前的状态。

  • 看到第四行64d2d5e HEAD@{3}: merge feature-A: Merge made by the 'recursive' strategy.,是feature-A特性分支合并后的状态，对应的哈希值是64d2d5e，我们将HEAD、暂存区、工作树恢复到这个时间点的状态。

2. 消除冲突

3. git commit –amend（修改提交信息）

上次提交commit，信息记为”Fix conflict”，不妥，其实是fix-B分支的合并，解决冲突只是过程之一，标记不妥，修改一下

执行上述命令后，编辑器启动,出现以下内容

Fix conflict

# with ‘#’ will be ignored, and an empty message aborts the commit.
#
# Date: Fri Jul 31 10:49:28 2015 +0800
#
# On branch master
# Changes to be committed:
# modified: README.md
